# Gameplay 音频系统

## 资源与场景入口

- 音频资源统一放在 `Assets/BGMSFX/`。
- `Zhuan` 场景根节点的 `Gameplay Audio` 是局内音频入口。
- 节点上保留三个 `AudioSource`，分别播放 BGM、飞船运行底噪和可重叠的事件音效。
- `GameplayAudioController` 中的具体音频均为 Inspector 引用，后续可以直接替换素材或调整三类音量。

## 当前触发规则

| 事件 | 播放规则 |
|---|---|
| 进入 Zhuan | 循环播放 `BGM(MEOW 版)` 和 `飞船运行01` |
| 成功或失败结算 | 停止飞船运行底噪，BGM 继续播放到离开场景 |
| 出现可交互操作提示 | 播放 `信息提示01`，提示持续存在时不会重复播放 |
| 玩家开始操作船锚台 | 随机播放 `操作炮台01~02` |
| 玩家成功拾取物品 | `材料拾取01~02` 与 `Meow01~04` 各随机一个并同时播放 |
| 飞船发生物理碰撞 | 随机播放 `飞船撞击01~02`，带短冷却防止同帧堆叠 |
| 物品成功 PutIn | 随机播放 `投放材料01~02` |
| 船锚成功发射 | 随机播放 `长锚01~02` |
| 船锚抓到任意物品 | 播放 `信息提示02` |

## UI 按钮音效

`ButtonSelectionAudio` 可以直接拖到任意带 `Button` 的 UI 对象上。组件监听该按钮的 `onClick`，鼠标点击或标准手柄 Submit 真正触发按钮功能时播放 `选择01`；MainMenu 自定义的 South/East 手柄快捷键也会主动通知此组件。音效使用跨场景的一次性播放对象，开始游戏切换场景时不会被截断。同一按钮同一帧只播放一次，避免按钮回调和快捷键同时触发造成叠音。`选择01` 已作为脚本默认引用配置，也可以在 Inspector 中单独替换。

## 角色选择音效

- 已加入玩家用左摇杆或方向键成功切换到另一名角色时播放 `选择01`。摇杆需要回到释放阈值后才能再次切换，因此不会持续连播。
- 未加入的手柄按 South 并成功加入时播放 `选择02`。
- 已加入玩家按 South 成功确认角色时播放 `选择02`；重复确认或角色已被占用时不播放。
- 所有玩家确认后，P1 按 North 进入 Gameplay 时播放 `选择02`。这次音效使用跨场景一次性音源，加载 `Zhuan` 时不会被截断。

## 代码接入约定

玩法脚本只调用 `GameplayAudioController` 的静态事件方法，不直接查找或加载音频文件。随机选择、通道播放和结算停止逻辑集中在控制器内。场景卸载时实例会自然销毁；静态引用通过 `SubsystemRegistration` 重置，以兼容关闭 Domain Reload 的编辑器设置。

当前项目尚无独立的“开始弹窗”流程，因此 BGM 和飞船底噪都从 `Zhuan` 场景正式启动时开始。后续增加开场弹窗后，可把 BGM 的开始调用前移到弹窗显示事件，并把底噪保留在正式开始事件。
